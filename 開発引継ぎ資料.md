## 0. これは何か

教材（画像 / PDF / 貼り付けテキスト）を読み込ませると、**Claude API** が問題を自動生成し、出題・採点・復習・成績管理まで行う学習アプリ。フレームワーク不使用の素のJS。1ファイルに HTML/CSS/JS を全部内蔵。

- 動作モード2種：
  - **standalone**（単体HTML）：利用者の Anthropic APIキーでブラウザから直接 API 呼び出し（画像・PDF対応）。
  - **widget**（Claudeアーティファクト内）：`window.claude.complete`（テキストのみ）。`IS_WIDGET` で分岐。
- 既定モデル `claude-sonnet-4-6`。廃止時は現行へ自動追従（`resolveActiveModel`）。

## 1.5 本セッションでの追加・修正（v5公開後の最新／引継ぎ最重要）

> v5公開後に追加・修正した内容。**上の §1 の一部（特に「穴埋め自己修復」）はここで方式変更されている**ので、矛盾時はこちらが正。

### 論理バグ修正
| 箇所 | 内容 |
|---|---|
| 並べ替え採点 | `submitOrderAnswer`/`renderOrderQuestion`：**重複する文言の項目**があると正しく並べても不正解になる不具合を、インデックス比較→**値（テキスト）比較**に修正。 |
| 選択肢 | `normalizeQuestions`：`correct` が選択肢数の範囲外だと正解が選べないため、範囲外は 0 に補正。 |
| 難易度ピップ | `showQuizQuestion`：難易度順モードは適応OFFで `currentDifficulty` が固定なので、その場合は各問の `q.difficulty` を表示。 |
| 画像形式 | `handleFiles`/`SUPPORTED_UPLOAD_TYPES`：Anthropic非対応のHEIC等をアップロード時に警告して除外（対応は PDF/JPEG/PNG/GIF/WebP）。UIのHEICタグも削除。 |

### 穴埋めの「数式の中の空欄」（行列など）対応【方式変更・重要】
- 旧 `normalizeFillMath`（数式を空欄位置で分割）は**行列 `\begin{bmatrix}` や `\frac` を壊す**ため**廃止**。
- 共通ヘルパー **`fillQuestionTextWith(q, fill)`** に統一（出題・振り返り `formatFillQuestionText`・印刷で共用）。数式の外/中を判定して置換するだけで、**数式は分割しない**。
- `renderFillQuestion`：**数式の外**の空欄は従来どおりインライン入力。**数式の中**の空欄は `\boxed{(n)}` プレースホルダで数式を丸ごと描画し、入力欄は番号付きで**下にまとめて**表示。
- 採点はDOM順ではなく **`dataset.idx`** で `q.blanks` と対応（順序ずれ防止）。

### 再開時のファイル復元（IndexedDB）
- アップロード上限を **3**（`MAX_FILES`）に統一。
- 生成成功時に元ファイルを **IndexedDB**（DB `aiquiz` / store `resume_files`）へ保存（`persistResumeFiles`）。`localStorage` では容量的にPDFを保持できないため。
- 「続きから再開」で読み戻し（`restoreResumeFiles`）→ 図の切り出し・出典表示が復活。**完了・破棄で削除**（`clearResumeFiles`、`clearQuizState` から呼ぶ）。ヘルパーは `idbOpen`/`idbRun`。
- プライバシー：中断を放置している間は教材がブラウザ内に残る旨を使い方モーダルに明記。

### 記述（text）UI
- `renderTextQuestion`：回答欄を入力量に応じて**自動リサイズ**（上限＝画面の70%）。
- 行列が絡む記述では**書き方ヒント**（`[1 2; 3 4]` 表記）を表示（`involvesMatrix` で判定）。

### 数式テンプレートビルダー（旧・行列ビルダーを一般化）
- 1つのモーダルでテンプレートを選んでスロットを埋め、**KaTeXライブプレビュー**を見ながら、カーソル位置に素のテキスト表記を挿入する入力補助。**採点は賢くならない**（文字列一致＋AI同値のまま）＝入力の手間を減らすだけ。
- テンプレート：**行列／積分／総和／総積／極限／分数／場合分け**（`TB_LIST`、fields型は `TB_FIELDS`、行列と場合分けは可変サイズ）。生成例：`[1 2; 3 4]`／`∫[0,1] x^2 dx`／`Σ[i=1,n] i^2`／`Π[k=1,n] k`／`lim(x→0) (sin x)/x`／`(a+b)/(2)`／`{ x (x>0のとき); -x (x<0のとき) }`。
- 関数：`openTemplateBuilder(initial?)`/`closeTemplateBuilder`/`selectTemplate`/`renderTemplate`/`buildTemplateNotation`/`buildTemplatePreviewTex`/`insertTemplate`。旧名 `openMatrixBuilder`/`closeMatrixBuilder` は後方互換エイリアスで残置。
- 数式キーボードのヘッダー「⊞ 数式テンプレート」と、行列系記述のヒントボタンから開ける。
- 開いている間はキーボードを**モーダル前面**（CSS `above-modal`）に出し、**フォーカス中のスロットへ記号入力**（θ・^ 等）できる。挿入先は `tbTarget`（元の回答欄）。

### その他の小修正
- `mathToHtml`：`$...$` の中身に日本語（ひらがな・カタカナ・漢字）があれば数式描画せず素表示（「価格は$5と$10」の誤レンダリング対策）。
- `clearFigureCaches`：不要になったオブジェクトURLを `revokeFileUrls` で解放（メモリリーク防止）。

### β機能（実験的・**既定オフ**）
- ヘッダー「β機能」→ モーダルで特徴説明＋トグル（`aiquiz_beta_draw` / `aiquiz_beta_graph`）。ONにすると「回答形式」に該当β形式が出現し、`buildSystemPrompt` にその型仕様が加わる。`loadBetaSettings`/`onBetaToggle`/`applyBetaToFormatOptions`。
- **`draw`（自由描画＋AI採点）**：`<canvas>`＋Pointer Eventsで手描き → 画像をVisionで採点（`gradeDrawAnswer`、standalone/APIキー時のみ）。画像を送れない環境は模範解答と見比べる**自己採点**（`showDrawSelfGrade`）。`renderDrawQuestion`/`submitDrawAnswer`/`drawGradeAvailable`。
- **`graph`（格子点プロット）**：SVGグリッドの整数格子点をクリックで打点 → **決定論で採点**（`graphCorrect`：points=順不同／polyline=順序あり・逆順可）。`renderGraphQuestion`/`drawGraph`/`svgClientToLattice`/`submitGraphAnswer`。AI不要・追加料金なし。
- どちらも `QUESTION_TYPES` に追加し、normalize/render/採点/結果/印刷に統合。**生成品質は粗く、draw採点の精度は目安**なので本格使用は非推奨（だからβ）。

## 1.6 本セッション（スマホ対応・配布準備）での追加・修正【最新・最重要】

> §1.5 のさらに後。スマホ実機（Android / Brave）で #1〜#6 を確認済み。矛盾時は本節が最優先。
> 方針：**機能は変えずCSS中心**。挙動変更が要る所だけ最小限のJSを足し、既存ロジック・採点は不変。

### スマホ対応
- **iOS自動ズーム防止**：`@media (max-width:540px)` で文字入力系（input/textarea/**select**＋`.fill-input`/`.text-answer`/`.table-input`/`.mb-cell`）を `font-size:16px !important`。16px未満だとフォーカス時にSafariが拡大するため。`#api-key` はインライン14pxを持つので `!important` 必須。selectもタップでズームするので対象。
- **ヘッダーのハンバーガーメニュー化**：モバイルではナビ（`.header-nav#header-nav`）を畳み `☰`（`#hamburger-btn`）で開閉。`toggleHeaderMenu`/`closeHeaderMenu`。項目タップ・外側タップで自動クローズ。PCは従来の横並び（`☰` は `display:none`）。
- **モーダルの背面スクロールロック**：全 `.modal-overlay` の `.show` を MutationObserver で監視し、開いている間 `body.modal-open{overflow:hidden}`（`syncBodyScrollLock`/`watchModalsForScrollLock`）。β機能モーダルだけスクロール容器が無く中身が見切れていたのを修正（中身 div に `flex:1;overflow-y:auto`）。連鎖防止に各スクロール領域へ `overscroll-behavior:contain`。
- **数式キーボード×OSキーボードの二重表示抑制**：数式キーボードを開いている間、回答欄（`[data-mathmanaged]`）の `inputmode='none'` でOS側ソフトキーボードを抑制（`setOSKeyboardSuppressed`、`registerMathInput`/`toggleMathKeyboard` で制御）。閉じれば解除して通常入力（数字・英字・日本語IME）に戻る。**タッチ端末のみ**開いた瞬間に `blur()` で既出のOSキーボードを消す（PCはカーソル維持のため blur しない）。※採点は送信ボタンのみで走り、Enter送信ハンドラは無いのでIME変換確定での誤採点は起きない。
- **数式キーボードに数字行を追加**：タブ非依存の常時表示行（`1234567890.`）。`.mk-numrow#mk-numrow`＋`renderNumRow`（`buildMathKeyboard` で一度描画）。キー生成は `createMathKey` に共通化。記号入力中に数字を打つためキーボードを閉じなくて済む。
- **学習履歴ダッシュボードの狭幅崩れ**：スコア推移バーが枠外へはみ出す→`.dash-bars{overflow-x:auto}`＋`.dash-bar-wrap{min-width:22px}` で枠内スクロール。トピック名が1文字に潰れる→`.topic-bar-track`/`.topic-pct` の固定幅を縮めて名前に領域を回す。
- **カスタムドロップダウン**：ネイティブ `<select>` のポップアップはアニメ不可なので、6つの設定 select を自前UI（`.cdd`）に置換し開閉をふわっとアニメ化。**本物の `<select>` はDOMに残して値の源泉**とし、選択時に `change` を発火→既存ロジック・採点は不変。`overflow:hidden` の祖先（詳細設定の `.guide-box`）に切られないよう `position:fixed` で配置＋下端なら上開き。キーボード操作・ARIA 対応。関数：`enhanceSelect`/`initCustomDropdowns`/`closeAllDropdowns`。

### バグ修正
- **AI作成図（figure_svg）が消える**：`viewBox` だけで `width`/`height` 属性が無いSVGがレイアウト上 **0×0 に潰れて**いた。`sanitizeSVG` で viewBox から寸法を補い、描画要素が無い空SVGは `null`（ボックス・キャプションごと非表示）。
- **出典PDFがスマホで表示できない**：`<iframe>` 内PDFはモバイルブラウザで描画不可。`openSourceModal` を **pdf.js で該当ページを画像化**して表示する方式に変更（`renderPdfPage` を再利用）。失敗時は「PDF全体を別画面で開く」リンク（`makeSourceOpenLink`）にフォールバック。画像出典は従来どおり。

### PWA / ホーム画面追加
- head に `theme-color`／Apple用メタ（`apple-mobile-web-app-*`、`apple-mobile-web-app-title`）／`apple-touch-icon`。実行時 `setupPwa()` がアイコンSVGを canvas で PNG 化（180/192/512、`rasterizeIcon`）し apple-touch-icon を PNG に差し替え＋**Blob で manifest を注入**（単一HTMLのまま）。`addToHomeScreen` は standalone 検出（`isStandalone`）→ `beforeinstallprompt`（ワンタップ）→ 手順モーダルの順。モーダルに端末別ヒント（`showInstallHint`）。
- **制約（重要）**：`file://`／`content://`（ファイルマネージャ起動）では**安定したURLが無い**ため、ホーム画面追加・自動インストールは効かない。実際に効かせるには **https でホスティング**（GitHub Pages 等）が必要。入れた manifest／メタは配信時に有効化される。
- **→ 解決済み**：GitHub Pages（https）で配信し、**公開URLでの手動「ホーム画面に追加」を実機（Android）で確認済み**（緑Aアイコン＋AIQuiz名＋standalone）。なお完全自動のワンタップ・インストールは Service Worker 無し＋manifestをBlob注入のため出ないことがある（手動追加は可）。ワンタップ自動／オフラインまで欲しければ静的 `manifest.webmanifest`＋Service Worker を足す（任意）。

- 公開ライセンスは **MIT**（`LICENSE`）。同梱している **KaTeX（MIT）とそのフォント（SIL OFL 1.1）** の帰属を、HTML内コメントと `THIRD-PARTY-LICENSES.md` に明記。**pdf.js（Apache 2.0）はCDN読み込みで非同梱**。

## 2. 画面の流れ

```
[アップロード] 教材・条件・APIキー・採点設定
   │ startQuiz()
   ▼
[ローディング] 生成中
   │ generateQuestions() → normalizeQuestions()
   ▼
[クイズ] 1問ずつ出題・回答・解説   showQuizQuestion()
   │ nextQuestion() / quitQuiz()（中断）
   ▼
[結果] 正答率・振り返り・印刷/保存/復習導線   showResults()
[ダッシュボード] 学習履歴   openDashboard()
```
モーダル：利用規約（`openNoticeModal`）・使い方（`openHelpModal`）・出典（`openSourceModal`）・AI判定の説明（`showAiEqInfo`）・ホーム画面追加（`addToHomeScreen`）。

## 4. 出題データの“契約”（AIが返すJSON / 保存セットの形）

共通：`type, question, explanation, difficulty(easy|medium|hard), topic, source{file,page,section,quote}`、任意 `figure_svg`（SVG文字列）/ `figure_ref{file,page,bbox[0..1]}`。

| type | 必須/特有フィールド |
|---|---|
| `choice` | `choices[]`, `correct`(索引) |
| `sort` | `items[]`, `categories[]`, `answer[]`(各itemの正解カテゴリ索引) |
| `order` | `items[]`（**正解順**で格納、表示時にシャッフル） |
| `fill` | `question`内に `___`、`blanks[]`、任意 `accept[][]`、任意 `strict`(真偽) |
| `table` | `rows[][]`(全セル埋めた状態)、任意 `headers[]`、`blanks[{r,c,accept?}]`、任意 `strict` |
| `text` | `model_answer`, `keywords[]` |
| `draw`(β) | `model_answer`(描くべき図の説明＝採点基準), `keywords[]`(採点要点)。Visionで採点。βがONの時のみ生成。 |
| `graph`(β) | `grid{xmin,xmax,ymin,ymax}`(整数), `points[[x,y]]`(正解の格子点), `mode`("points"=順不同/"polyline"=順序あり)。決定論採点。βがONの時のみ生成。 |

ルール要点：**表示テキスト**（問題文・選択肢・解説・表セル等）は `$...$` の LaTeX。**回答値**（fillの`blanks`/`accept`、tableの空欄セル）は `$` を使わず素のUnicode。語学で綴り・大小文字が重要な問題は `strict:true`。

## 6. 既知の制約（バグではなく設計上の割り切り）

- **ローカルファイル（file://）ではAPIが遮断されることがある** → `起動.bat`/`server.ps1` で localhost 経由起動。
- **ホーム画面追加・PWAインストールは `file://`／`content://` では不可**（安定したURLが無いため、ブラウザがショートカットを作れない）。実際に効かせるには **https でホスティング**（GitHub Pages 等）が必要。head のメタ・実行時注入の manifest は配信時に有効化される。§1.6 参照。
- **AI任せのソフトな歯止め**（ヒント漏れ防止・図に答えを写さない・LaTeX/Unicode分離・`strict`自動付与）はプロンプト依存で**100%保証ではない**。数回の生成で要観察。
- **図の内容の正しさは自動検証不能**（「参考・誤りがあり得ます」と明示）。
- **数式同値**は `+ - ×`（積・和）の順序入替のみ。割り算・括弧・累乗を含む式は**完全一致のみ**（ただし上付き→`^`、波/角括弧→丸括弧、`x²`=`x^2` 等の正規化で多くは吸収）。上付きは `^n`、下付きは素の数字に畳む（`x²`と`x₂`は別物）。
- **クロスOSの“ダブルクリック起動”は1ファイルでは不可**（OSごとに拡張子/実行権限の仕組みが違う）。`起動.bat`はWindowsはダブルクリック、Mac/Linuxはターミナル実行 or `.command`にリネーム。
- **記述・β描画のAI採点は「目安」**。特に手描き図（β `draw`）の精密な読み取りはVisionが苦手で誤判定しやすい。β `graph` は決定論なので正確だが、AIが格子点で表せない問題を作ると解けないことがある（β＝生成品質は要観察）。
- **数式中に日本語を入れると素表示になる**：通貨記号 `$5と$10` の誤レンダリング回避のため、`$...$` の中身に日本語があると数式描画しない。プロンプトは「数式に日本語を混ぜない」方針なので通常は問題なし。
- **再開時の図・出典は IndexedDB が使える環境前提**。`file://` 直開きでは一部ブラウザで不安定なので、`起動.bat`（localhost）経由推奨。

## 7. 残作業

> **PC・スマホ・ホスティング・ホーム画面追加（手動）まで実機確認済みで一区切り**（§1.6）。下記は「やってもやらなくても可」の任意項目のみ。

- **任意：ワンタップ自動インストール／オフライン対応**：現状は手動「ホーム画面に追加」のみ確認済み。完全自動のインストールプロンプトやオフライン動作が欲しければ、静的 `manifest.webmanifest` ＋ Service Worker を追加（ホスティング済みなので実現可能）。
- **任意：数式テンプレートビルダーの二重キーボード**：ビルダーのモーダル内スロットは `data-mathmanaged` 対象外なので、そこでは数式キーボードとOSキーボードの二重表示が残りうる（細かいエッジケース）。
- **任意：リポジトリ仕上げ**：README/About欄/トピック（topics）の充実、コミットメッセージの整備など。
- **β機能の作り込み**：`draw`/`graph` は「動く」段階で「こなれている」段階ではない。生成プロンプトの精度、draw採点の安定性、graphの問題妥当性（格子点で表せるか）は実使用での観察・調整が必要。本格運用は見送り中（既定オフ）。Pointer Events採用なのでスマホ・スタイラスとの相性は要観察。
- **微細なバグ**は今後の実使用で炙り出す段階。機能・ドキュメントとも現状で完了。
